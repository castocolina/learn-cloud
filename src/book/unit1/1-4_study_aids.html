Question

What are the key principles of REST (Representational State Transfer)?

Answer

Stateless: Each request contains all needed information. Resource-based: URLs identify resources. HTTP methods: Use GET, POST, PUT, DELETE appropriately. Cacheable: Responses can be cached. Uniform interface: Consistent API design.

Question

What is the difference between HTTP GET and POST methods?

Answer

GET: Retrieves data, should be idempotent and safe, parameters in URL. POST: Creates new resources, not idempotent, data in request body, can have side effects.

Question

What are HTTP status codes and what do the main ranges represent?

Answer

2xx: Success (200 OK, 201 Created). 3xx: Redirection (301 Moved). 4xx: Client errors (404 Not Found, 400 Bad Request). 5xx: Server errors (500 Internal Server Error).

Question

What is JSON and why is it preferred for API communication?

Answer

JSON (JavaScript Object Notation) is a lightweight data-interchange format. It's human-readable, language-independent, has excellent parsing support, smaller than XML, and natively supported by web browsers.

Question

What is the purpose of API versioning and what are common strategies?

Answer

API versioning manages changes without breaking existing clients. Common strategies: URL versioning (/v1/users), Header versioning (Accept: application/vnd.api+json;version=1), Query parameters (?version=1).
