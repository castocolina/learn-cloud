Question

What are the three levels of the Testing Pyramid and their relative proportions?

Answer

1. Unit Tests (Many, Fast): Test individual functions or methods in isolation.
2. Integration Tests (Fewer, Medium Speed): Verify interactions between components.
3. End-to-End Tests (Few, Slow): Simulate real user scenarios across the entire system.

Question

What is the purpose of mocking in unit tests?

Answer

Mocking allows you to replace external dependencies (like databases, APIs, or services) with fake objects that return predictable responses. This ensures tests are isolated, fast, and don't depend on external systems being available.

Question

What pytest decorator is used to parameterize test functions?

Answer

@pytest.mark.parametrize allows you to run the same test function with multiple sets of arguments. This is useful for testing edge cases and different input combinations efficiently.

Question

What is test coverage and why is it important?

Answer

Test coverage measures the percentage of your code that is executed during tests. While 100% coverage doesn't guarantee bug-free code, it helps identify untested areas and ensures critical paths are validated. Tools like pytest-cov can generate coverage reports.

Question

What is the difference between unit tests and integration tests?

Answer

Unit Tests: Test individual components in isolation, using mocks for dependencies. They're fast and provide precise feedback.

Integration Tests: Test how multiple components work together, often with real dependencies like databases. They're slower but provide more confidence in system behavior.

Question

What is Test-Driven Development (TDD) and what are its three phases?

Answer

TDD is a development approach where you write tests before writing the implementation code. The three phases are:
1. Red: Write a failing test
2. Green: Write minimal code to make the test pass
3. Refactor: Improve the code while keeping tests green

Question

What are pytest fixtures and how are they useful?

Answer

Pytest fixtures are reusable setup and teardown code for tests. They can provide test data, database connections, or mock objects. Fixtures help eliminate code duplication and ensure consistent test environments. They're defined with the @pytest.fixture decorator.

Question

Why is testing particularly important for cloud-native applications?

Answer

Cloud-native applications are distributed, often involving multiple services, containers, and external dependencies. Comprehensive testing ensures reliability across different environments, helps catch integration issues early, and provides confidence when deploying to production via CI/CD pipelines.
