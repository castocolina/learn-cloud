Unit 9-1: Study Aids - E-Commerce Microservices Project

Review key concepts from the Python-based E-Commerce Microservices capstone project. These flashcards cover architecture patterns, FastAPI implementation, and cloud-native deployment strategies.

Question

What are the three core microservices in the e-commerce platform and their primary responsibilities?

Answer

1. Product Service (FastAPI): Manages product catalog with RESTful endpoints for CRUD operations.
2. Order Service (FastAPI): Handles order creation and publishes OrderCreated events to message queue.
3. Payment Worker: Background service that consumes order events and processes payments asynchronously.

Question

How does the system implement event-driven architecture for order processing?

Answer

When the Order Service receives a POST request, it creates an order in PostgreSQL and publishes an "OrderCreated" event to a message queue. The Payment Worker consumes these events asynchronously, simulates payment processing, and updates the order status back in the database.

Question

What role does HashiCorp Vault play in the microservices architecture?

Answer

Vault serves as the centralized secrets management system, securely injecting database credentials into all three services (Product Service, Order Service, and Payment Worker). This eliminates hardcoded secrets and provides secure, auditable access to sensitive configuration data.

Question

What infrastructure components are managed by Terraform in this project?

Answer

Terraform manages the complete infrastructure stack: EKS Cluster for container orchestration, PostgreSQL databases for both services, Message Queue for event processing, and HashiCorp Vault for secrets management. This Infrastructure as Code approach ensures consistent, reproducible deployments.

Question

How does Kubernetes Ingress enhance the system's architecture?

Answer

Kubernetes Ingress acts as the single entry point for external traffic, providing load balancing, SSL termination, and routing to the appropriate microservices. It eliminates the need for individual service load balancers and provides centralized traffic management with path-based routing.
