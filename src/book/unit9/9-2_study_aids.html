Unit 9-2: Study Aids - Go Real-Time Analytics Pipeline

Review key concepts from the Go-based Real-Time Analytics Pipeline capstone project. These flashcards cover gRPC streaming, Apache Kafka, time-series databases, and distributed system architecture.

Question

What are the main components in the real-time analytics pipeline and their data flow?

Answer

Data Flow: Data Source → Ingestion Service (gRPC) → Apache Kafka → Processing Service → Time-Series DB → Query Service (REST API). Each component handles a specific aspect: ingestion via gRPC streams, event publishing to Kafka, stream processing for aggregations, and query serving via REST API.

Question

Why is gRPC chosen for the data ingestion interface instead of REST?

Answer

gRPC provides bidirectional streaming capabilities essential for real-time data ingestion, with lower latency and better performance than REST. It supports efficient binary serialization (Protocol Buffers), built-in load balancing, and maintains persistent connections ideal for high-throughput data streams.

Question

What role does Apache Kafka play in the analytics pipeline architecture?

Answer

Kafka serves as the central event streaming platform, decoupling the Ingestion Service from Processing Service. It provides durable message storage, horizontal scaling, fault tolerance, and replay capabilities. This enables the system to handle high-volume data streams while maintaining reliability and processing guarantees.

Question

Why is a Time-Series Database specifically chosen for storing aggregated analytics data?

Answer

Time-Series DBs are optimized for timestamp-indexed data with high write throughput and efficient range queries. They provide built-in compression, retention policies, and aggregation functions essential for analytics workloads. This makes them ideal for IoT data, metrics, and real-time analytics scenarios requiring temporal data operations.

Question

How does the Query Service provide access to processed analytics data?

Answer

The Query Service exposes a REST API that allows users and applications to retrieve aggregated analytics data from the Time-Series Database. It handles query optimization, data formatting, and provides a standard HTTP interface that can be easily consumed by dashboards, applications, and monitoring tools.
