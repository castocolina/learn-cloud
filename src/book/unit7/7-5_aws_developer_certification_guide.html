Unit 7.5: AWS Developer Certification Guide

Earning the AWS Certified Developer - Associate (DVA-C02) certification is a valuable way to validate your skills and knowledge of AWS services. This unit has covered many of the core serverless concepts that are heavily featured on the exam. This topic provides a focused guide to the key serverless concepts you should master for the exam and offers study tips for success.

Disclaimer: This is a study guide, not an exhaustive list of all exam topics. Always refer to the official AWS Certified Developer - Associate Exam Guide for the most current information.

Key Serverless Concepts for the Exam

The DVA-C02 exam emphasizes practical knowledge of developing, deploying, and debugging cloud-native applications on AWS. Serverless is a major component of this.

1. AWS Lambda

  • Execution Model: Understand the entire lifecycle: invocation types (synchronous, asynchronous, event source mapping), the event object, the context object, and what happens during a cold vs. warm start.
  • Permissions: Know that Lambda functions use an IAM Role (Execution Role) to get permissions to access other AWS services (e.g., read from S3, write to DynamoDB).
  • Concurrency: Be able to explain the difference between on-demand and provisioned concurrency, and know what Lambda throttling is and how to use reserved concurrency to prevent it.
  • Versions and Aliases: Understand how to use versions to create immutable snapshots of your code and aliases (e.g., `dev`, `prod`) to point to specific versions for safe deployments like blue/green.
  • Destination and DLQs: Know how to configure a Dead-Letter Queue (DLQ) using SQS or SNS to handle failed asynchronous invocations.

2. Amazon API Gateway

  • Integrations: Understand the difference between Lambda proxy integration (where the entire request is passed to Lambda) and non-proxy (custom) integration. The exam heavily favors proxy integration.
  • Authentication & Authorization: Be familiar with the different ways to secure an API, including IAM roles, Lambda authorizers (formerly Cognito authorizers), and resource policies.
  • Stages: Know that you deploy an API to a "stage" (e.g., `dev`, `v1`, `prod`) and how to use stage variables.

3. Amazon DynamoDB

  • Core Concepts: Understand tables, items, primary keys (partition key, sort key), and attributes.
  • Capacity Modes: Know the difference between On-Demand (pay-per-request) and Provisioned (specify read/write capacity units) modes.
  • Consistency Models: Understand the difference between strongly consistent reads and eventually consistent reads (the default).
  • DynamoDB Streams: Know that Streams capture a time-ordered sequence of item-level modifications and can be used as an event source to trigger Lambda functions.

4. Amazon SQS & SNS

  • SQS (Simple Queue Service): Understand its role in decoupling services. Know the difference between standard queues (at-least-once delivery, best-effort ordering) and FIFO queues (first-in, first-out delivery, exactly-once processing). Know what a DLQ is for.
  • SNS (Simple Notification Service): Understand its publish/subscribe model. Know that SNS can deliver messages to multiple subscribers, including SQS queues, Lambda functions, and HTTP endpoints.
  • Fan-Out Pattern: Be able to describe the common pattern of using SNS to fan out a single message to multiple SQS queues, which are then processed by Lambda functions.

5. Deployment & Tooling

  • AWS SAM: Be familiar with the structure of a template.yaml file and the purpose of the sam build, sam package, and sam deploy commands.
  • AWS CloudFormation: Understand that SAM is an extension of CloudFormation. You should know the basic structure of a CloudFormation template and how it deploys resources as a "stack".

Study Tips for Success

  1. Hands-On is Non-Negotiable: You cannot pass this exam by only reading. Use the AWS Free Tier to build small applications using Lambda, API Gateway, and DynamoDB. Deploy them using AWS SAM.
  2. Know the AWS SDK: For the DVA-C02, you need to know the names and basic usage of key SDK calls. For example, for DynamoDB, you should know `GetItem`, `PutItem`, `UpdateItem`, and `Query`. You don't need to memorize every parameter, but you should know what each call does.
  3. Understand IAM Roles Deeply: A significant portion of questions will involve permissions. Always remember that AWS services use IAM Roles to get the permissions they need to interact with other services.
  4. Read the Official Exam Guide and Sample Questions: The official AWS materials are the best source of truth for the exam's scope. Review them carefully.
  5. Take Practice Exams: Use reputable practice exams (e.g., from Stephane Maarek, Jon Bonso) to gauge your readiness and identify weak areas. Analyze why you got questions wrong, don't just memorize the answers.
