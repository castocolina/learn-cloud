Unit 7-1: Study Aids - The Serverless Spectrum

Review key concepts of serverless computing, including the evolution from IaaS to FaaS and container services.

Question

What is the core philosophy of "serverless" computing?

Answer

Serverless computing abstracts away infrastructure management. Developers focus on writing application logic, and the cloud provider is responsible for provisioning, scaling, and maintaining the servers that run the code.

Question

What is AWS Lambda?

Answer

AWS Lambda is a Function as a Service (FaaS) offering. It allows you to run code in response to events without provisioning or managing any servers. You are billed only for the compute time you consume.

Question

What is AWS Fargate?

Answer

AWS Fargate is a serverless compute engine for containers. It allows you to run containerized applications on ECS or EKS without managing the underlying EC2 instances or nodes of the cluster.

Question

What is a key difference in the cost model between Lambda and Fargate?

Answer

Lambda is billed per request and per millisecond of execution, making it ideal for spiky traffic. Fargate is billed per second for the vCPU and memory resources requested by your container, making it more cost-effective for consistent, long-running workloads.

Question

When would you choose Fargate over Lambda?

Answer

You would choose Fargate when you need to run an existing containerized application, when you have a long-running service, or when your application requires more control over the execution environment than Lambda's function model allows.
