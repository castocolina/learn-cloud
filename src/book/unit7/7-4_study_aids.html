Study Aids: Serverless Deployment & Tooling

Key Takeaways
  • IaC is Essential for Serverless: Manually managing the many interconnected resources of a serverless application is not scalable. An Infrastructure as Code (IaC) framework is necessary to define, version, and reliably deploy the entire application stack.
  • Serverless Framework: The Multi-Cloud Option: The Serverless Framework is a popular, open-source tool known for its multi-cloud support (AWS, Azure, GCP) and its rich ecosystem of plugins. It uses a serverless.yml file for configuration.
  • AWS SAM: The Native Option: The AWS Serverless Application Model (SAM) is AWS's own open-source framework. It is an extension of CloudFormation, uses a template.yaml file, and offers excellent integration with AWS services and powerful local testing capabilities via sam local.
  • Both Use CloudFormation: On AWS, both frameworks ultimately translate their configuration into an AWS CloudFormation template to provision the necessary resources in a reliable, transactional manner.
  • Choosing a Framework: The choice depends on your needs. Use the Serverless Framework for multi-cloud requirements or its extensive plugin ecosystem. Use AWS SAM for a native AWS-centric experience and strong local testing capabilities.

Question

Why is Infrastructure as Code (IaC) critical for serverless applications?

Answer

Because serverless applications are composed of many small, interconnected resources (Lambda, API Gateway, IAM Roles, etc.). IaC allows you to define, version, and reliably deploy this entire stack as a single unit, which is unmanageable to do manually.

Question

What is the name of the configuration file used by the Serverless Framework?

Answer

serverless.yml

Question

What is the name of the configuration file used by the AWS Serverless Application Model (SAM)?

Answer

template.yaml

Question

What is a key advantage of AWS SAM for local development?

Answer

The sam local command set, which allows you to invoke functions and run a local API Gateway in a Docker container that simulates the real AWS Lambda environment, enabling rapid local testing and debugging.

Question

What underlying AWS service do both the Serverless Framework and AWS SAM use to deploy resources on AWS?

Answer

AWS CloudFormation. Both frameworks translate their respective YAML configurations into a CloudFormation template and use it to create, update, or delete the application stack.
