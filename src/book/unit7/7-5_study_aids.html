Study Aids: AWS Developer Certification Guide

Key Takeaways
  • Focus on Core Services: The AWS Certified Developer - Associate exam heavily emphasizes core serverless services: Lambda, API Gateway, DynamoDB, SQS, and SNS.
  • IAM is Everywhere: A deep understanding of how services get permissions via IAM Roles (especially Lambda Execution Roles) is critical for a large number of exam questions.
  • Practical Knowledge is Key: The exam tests your ability to develop, deploy, and debug. Rote memorization is not enough; you must have hands-on experience building with these services.
  • Know the Deployment Tools: Be familiar with the purpose and basic commands of AWS SAM (sam build, sam deploy) and understand its relationship to the underlying AWS CloudFormation service.
  • Understand Service Integrations: Know the common patterns, such as API Gateway with Lambda proxy integration, and using DynamoDB Streams or SQS as event sources for Lambda.

Question

How does an AWS Lambda function get permission to write to a DynamoDB table?

Answer

Through its IAM Execution Role. The policy attached to this role must grant the necessary permissions (e.g., dynamodb:PutItem).

Question

What is the difference between a Lambda alias and a version?

Answer

A version is an immutable, numbered snapshot of your function's code and configuration. An alias is a mutable pointer that can point to a specific version. Aliases are used to create stable endpoints (e.g., `prod`, `dev`) for different environments.

Question

What is the "fan-out" pattern and which two services are commonly used to implement it?

Answer

The fan-out pattern is when a single message is delivered to multiple subscribers for parallel processing. It is commonly implemented by publishing a message to an SNS topic, which then pushes the message to multiple SQS queues.

Question

What is the difference between a strongly consistent read and an eventually consistent read in DynamoDB?

Answer

A strongly consistent read returns the most up-to-date data, reflecting all successful prior writes, but has higher latency and cost. An eventually consistent read (the default) is faster and cheaper but might return slightly stale data a moment after a write.

Question

What is the purpose of a Dead-Letter Queue (DLQ) for an asynchronous Lambda invocation?

Answer

A DLQ (typically an SQS queue or SNS topic) is a destination where Lambda can send an event after it has failed all retry attempts. This prevents losing the event and allows for later analysis or reprocessing.
