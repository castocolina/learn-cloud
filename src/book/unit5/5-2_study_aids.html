Study Aids: SCA (Software Composition Analysis)

Key Takeaways
  • You Are What You Import: Modern applications are built on open-source dependencies. SCA is the process of identifying these dependencies and scanning them for known vulnerabilities (CVEs).
  • Supply Chain Risk: A vulnerability in a third-party library is a vulnerability in your application. Managing this software supply chain risk is a critical part of DevSecOps.
  • Tooling for Your Ecosystem: Use language-specific SCA tools. For Python, pip-audit or safety are common choices. For Go, the official tool is govulncheck.
  • Context is Key (Go): Go's `govulncheck` tool is particularly powerful because it only reports vulnerabilities in dependency code that your application actually calls, reducing false positives and alert fatigue.

Question

What is Software Composition Analysis (SCA)?

Answer

SCA is the automated process of identifying the open-source and third-party components used in an application and checking them for known security vulnerabilities.

Question

How does SCA differ from SAST?

Answer

SAST (Static Application Security Testing) analyzes the code you wrote yourself for vulnerabilities. SCA analyzes the third-party code (dependencies) you import for known vulnerabilities.

Question

What is the standard SCA tool for Python projects?

Answer

pip-audit, which is developed by the Python Packaging Authority (PyPA) and checks dependencies against the PyPI Advisory Database.

Question

What is a key advantage of Go's `govulncheck` tool?

Answer

It performs call graph analysis to determine if your application actually invokes the vulnerable code within a dependency. This reduces noise by only reporting vulnerabilities that are reachable and therefore pose a more direct risk.

Question

What is the "software supply chain"?

Answer

It refers to the entire ecosystem of components, libraries, and dependencies (both direct and transitive) that make up your application. Securing the supply chain means ensuring all these components are free from known vulnerabilities.
