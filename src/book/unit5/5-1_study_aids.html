Question

What is "Shifting Left" in the context of DevSecOps?

Answer

It is the practice of moving security checks and processes to the earliest possible point in the development lifecycle (i.e., further to the left on a typical project timeline diagram). This helps find and fix vulnerabilities when they are cheapest and easiest to address.

Question

What is Static Application Security Testing (SAST)?

Answer

SAST is a "white-box" testing methodology that analyzes an application's source code, bytecode, or binary for security vulnerabilities without executing the program.

Question

What is the standard SAST tool for Python, and what kind of issues can it find?

Answer

Bandit is the standard SAST tool for Python. It can find issues like hardcoded passwords, insecure use of subprocess, known insecure XML parsing, and weak cryptographic functions.

Question

What is the standard SAST tool for Go?

Answer

gosec is the standard SAST tool for Go. It analyzes the Go AST (Abstract Syntax Tree) to find security weaknesses.

Question

Why is it critical to integrate SAST tools into a CI/CD pipeline?

Answer

Integrating SAST into the CI/CD pipeline automates the security scanning process. It ensures that every code change (e.g., on every push or pull request) is automatically checked for vulnerabilities, providing rapid feedback to developers and preventing insecure code from reaching production.
