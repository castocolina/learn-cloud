Study Aids: Vulnerability Management

Key Takeaways
  • It's a Cycle, Not a Project: Vulnerability management is a continuous lifecycle of discovering, prioritizing, remediating, and verifying vulnerabilities.
  • Prioritization is Essential: You can't fix everything at once. Use the CVSS score as a starting point, but add business context (e.g., reachability, exposure) to determine the true risk of a vulnerability.
  • Define Clear SLAs: Service Level Agreements (SLAs) for remediation, based on vulnerability severity, create clear expectations for development teams and provide a measurable way to track security posture.
  • Automate Verification: Always re-scan after a fix is applied to verify that the vulnerability has been successfully remediated. This should be an automated part of the CI/CD process.

Question

What are the four stages of the vulnerability management lifecycle?

Answer

1. Discover: Find vulnerabilities using tools like SAST, SCA, and image scanners.
2. Prioritize: Assess the risk and urgency of each vulnerability.
3. Remediate: Fix the vulnerability.
4. Verify: Confirm the fix by re-scanning.

Question

What is CVSS?

Answer

The Common Vulnerability Scoring System (CVSS) is the industry standard for rating the severity of security vulnerabilities. It provides a numerical score from 0.0 to 10.0, which helps in prioritizing remediation efforts.

Question

Why is business context important for prioritizing vulnerabilities beyond just the CVSS score?

Answer

Context determines the actual risk. A high-severity vulnerability in a non-critical, internal service might be lower risk than a medium-severity one in a public-facing service that handles sensitive data. Factors include reachability, data sensitivity, and exposure.

Question

What is a remediation SLA in vulnerability management?

Answer

A Service Level Agreement (SLA) for remediation is a policy that defines the maximum amount of time a team has to fix a vulnerability after it has been discovered, usually based on its severity level (e.g., "Critical vulnerabilities must be fixed within 72 hours").

Question

What is the final step in the vulnerability management lifecycle after a patch has been applied?

Answer

Verification. The same tool that found the vulnerability should be re-run to confirm that the fix is effective and the vulnerability is no longer present.
