Study Aids: Kubernetes for Container Orchestration

Key Takeaways
  • Advanced Deployments Mitigate Risk: Kubernetes enables sophisticated deployment strategies like Blue/Green (instant switchover) and Canary (gradual rollout) to minimize the risk and impact of new releases.
  • Stateful Applications are First-Class Citizens: Running stateful applications is managed with StatefulSets, which provide stable network identities and storage, and PersistentVolumeClaims (PVCs), which allow applications to request durable storage.
  • Secure Your Network by Default: NetworkPolicies act as a firewall for pods, allowing you to implement a zero-trust network model by explicitly defining which services are allowed to communicate with each other.
  • Helm Simplifies Complexity: Helm is the package manager for Kubernetes. It simplifies the deployment and management of complex applications by bundling all their required Kubernetes resources into a single, versioned, and configurable package called a Chart.

Question

What is the main difference between a Blue/Green deployment and a Canary deployment?

Answer

In Blue/Green, you switch 100% of traffic to a new, identical environment all at once. In a Canary deployment, you release the new version to a small subset of users first and gradually increase traffic, monitoring for issues.

Question

What two key features does a StatefulSet provide that a Deployment does not?

Answer

1. Stable, unique network identifiers for each pod (e.g., pod-0, pod-1).
2. Stable, persistent storage that is tied to each pod's unique identity, ensuring a pod is always re-attached to its specific storage volume.

Question

What is the purpose of a Kubernetes NetworkPolicy?

Answer

To act as a firewall for pods. It allows you to control the ingress (incoming) and egress (outgoing) network traffic for a group of pods, securing the network within the cluster by enforcing rules about which services can communicate.

Question

What is Helm, and what is a Helm Chart?

Answer

Helm is the package manager for Kubernetes. A Chart is a package containing a collection of pre-configured Kubernetes resource templates that can be deployed, versioned, and managed as a single unit.

Question

What is the relationship between a PersistentVolume (PV) and a PersistentVolumeClaim (PVC)?

Answer

A PV is a piece of storage in the cluster, provisioned by an administrator. A PVC is a request for that storage made by a user or application. The PVC is "bound" to a suitable PV, abstracting the underlying storage details from the application.
