Study Aids: Spinnaker for Continuous Delivery

Key Takeaways
  • Spinnaker is for CD, not CI: Spinnaker is a specialized, multi-cloud Continuous Delivery (CD) platform designed for safe, high-velocity software releases. It complements, rather than replaces, CI tools like GitHub Actions.
  • Native Deployment Strategies: It provides native, first-class support for advanced deployment strategies like Red/Black (Blue/Green) and Canary, making them declarative and easy to manage.
  • Automated Canary Analysis is Key: Spinnaker's killer feature is Automated Canary Analysis (ACA), which automatically compares a new release against a baseline using real-time metrics to provide a data-driven assessment of its risk.
  • Use Them Together: The most effective pattern is to use GitHub Actions for CI (build, test, push artifact) and then trigger a Spinnaker pipeline to handle the complex CD part of the workflow (deploy, analyze, promote).

Question

What is the primary difference between Continuous Integration (CI) and Continuous Delivery (CD)?

Answer

CI focuses on automatically building and testing code after every change to ensure it integrates correctly. CD focuses on automatically and safely releasing the successfully tested code to production environments.

Question

In Spinnaker, what is a "Server Group"?

Answer

A Server Group represents a running, versioned instance of your application. In Kubernetes, this is typically a Deployment or ReplicaSet. In AWS, it's an Auto Scaling Group. Spinnaker manages the lifecycle of these server groups.

Question

What is Automated Canary Analysis (ACA) in Spinnaker?

Answer

It's a process where Spinnaker automatically deploys a small "canary" version of a new release, compares its performance metrics (from a tool like Prometheus) against the stable "baseline" version, and provides a score to determine if the release is safe to promote.

Question

Why is Spinnaker considered more powerful for deployments than a general-purpose CI tool like GitHub Actions?

Answer

Spinnaker is purpose-built for CD. It has native, first-class support for advanced deployment strategies, deep integrations with cloud providers (understanding their resources), and powerful features like Automated Canary Analysis that are very difficult to script in a CI tool.

Question

Describe the recommended workflow combining GitHub Actions and Spinnaker.

Answer

The GitHub Actions workflow handles the CI part: it builds the code, runs tests, and pushes a container image to a registry. Upon success, it triggers a Spinnaker pipeline, which then handles the CD part: deploying the new image using a safe strategy like Canary or Blue/Green.
