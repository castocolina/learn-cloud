Study Aids: Terraform for Infrastructure as Code

Key Takeaways
  • IaC is Foundational: Infrastructure as Code (IaC) is a core DevOps practice that brings versioning, automation, and reproducibility to infrastructure management. Terraform is the industry-standard tool for IaC.
  • State is Critical: Terraform relies on a state file to map your configuration to real-world resources. For team collaboration, using remote state with a locking mechanism (e.g., AWS S3 and DynamoDB) is non-negotiable to prevent conflicts.
  • Modules Enable Reusability: Terraform Modules are the primary way to create reusable, maintainable, and shareable infrastructure components. They are essential for managing complexity in large-scale environments.
  • The Core Workflow: The standard Terraform workflow consists of three commands: terraform init to prepare the environment, terraform plan to preview changes, and terraform apply to execute them.

Question

What is Infrastructure as Code (IaC)?

Answer

IaC is the practice of managing and provisioning infrastructure through machine-readable definition files (code), rather than through manual configuration or interactive tools. This enables versioning, automation, and reproducibility.

Question

What are the five core block types in HCL for Terraform?

Answer

1. terraform (configures Terraform itself)
2. provider (configures a specific provider like AWS)
3. resource (defines a piece of infrastructure)
4. variable (defines an input variable)
5. output (defines an output value)

Question

Why is using remote state with locking essential for teams using Terraform?

Answer

It provides a shared, central source of truth for the infrastructure's state. The locking mechanism (e.g., a DynamoDB table) prevents multiple team members from running terraform apply at the same time, which would corrupt the state and lead to infrastructure drift.

Question

What is the purpose of a Terraform module?

Answer

To package and reuse resource configurations. Modules are the primary way to create abstractions, organize code into logical components, and share common infrastructure patterns (e.g., a standardized S3 bucket or VPC).

Question

What are the three core commands in the standard Terraform workflow?

Answer

1. terraform init: Initializes the backend and downloads providers.
2. terraform plan: Shows a preview of the changes to be made.
3. terraform apply: Executes the changes to create, update, or delete infrastructure.
