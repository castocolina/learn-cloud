1.4 Study Aids: Core Backend Concepts

Use these flashcards to review the key concepts of backend data management in Python.

Python Dataclasses

What is the purpose of the @dataclass decorator in Python?

Answer

It automatically generates boilerplate methods for classes that primarily store data, such as __init__(), __repr__(), and __eq__(). This reduces code verbosity and simplifies class creation.

Direct Database Access

What is the most critical security risk when writing direct SQL queries, and how is it prevented?

Answer

The most critical risk is SQL Injection. It is prevented by using parameterized queries (passing parameters as a tuple to cursor.execute()), not by using string formatting to build SQL strings.

ORM (SQLAlchemy)

What is an Object-Relational Mapper (ORM) and what is its main advantage?

Answer

An ORM is a library that maps database tables to objects in code. Its main advantage is increased developer productivity by allowing developers to work with Python objects instead of writing raw SQL, which also reduces boilerplate code.

Redis

What is a primary use case for Redis in a cloud-native backend application?

Answer

The primary use case for Redis is caching. As a very fast in-memory key-value store, it is used to store the results of expensive operations (like database queries) to significantly reduce latency and database load.

MongoDB

What kind of data model does MongoDB use, and what is a key benefit of this model?

Answer

MongoDB uses a document-oriented data model, storing data in flexible, JSON-like BSON documents. A key benefit is its schema flexibility, which is ideal for applications with rapidly changing requirements or semi-structured data.
