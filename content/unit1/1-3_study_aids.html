1.3 Study Aids: Code Quality and Standards

Use these flashcards to review the key concepts of Python code quality.

PEP 8

What is the primary purpose of PEP 8?

Answer

PEP 8 is Python's official style guide. Its primary purpose is to improve the readability and consistency of Python code, making it easier for developers to collaborate and maintain projects.

Code Formatter (Black)

What is the main benefit of using an opinionated code formatter like Black?

Answer

It enforces a single, consistent style across the entire codebase, which eliminates all debates and arguments over code formatting. This allows developers to focus on logic and functionality instead of style.

Linter (Ruff)

What does a linter like Ruff do that a formatter like Black does not?

Answer

A linter analyzes the code for potential errors, bugs, and stylistic issues that go beyond formatting. It can detect issues like unused imports, undefined variables, and potential security vulnerabilities, which a formatter ignores.

Ruff's Key Advantage

What is the main advantage of using Ruff over older Python linters?

Answer

Ruff's key advantage is its speed. Written in Rust, it is significantly faster than traditional Python linters. It also combines the functionality of many tools into one, simplifying the toolchain.

Pre-commit Hooks

How do pre-commit hooks improve team workflow?

Answer

They automate code quality checks by running tools like Black and Ruff before a commit is created. This ensures that all code entering the repository adheres to the established standards, preventing errors and style inconsistencies from being committed in the first place.
