1.8 Study Aids: Testing Strategies

Use these flashcards to review the key concepts of testing in Python backend development.

Testing Pyramid

Describe the three main layers of the Testing Pyramid.

Answer

1. Unit Tests: Fast, isolated, test individual components.
2. Integration Tests: Verify interactions between components, slower than unit tests.
3. End-to-End Tests: Simulate user scenarios across the entire system, slowest and most brittle.

Mocking

When and why would you use mocking in your tests?

Answer

Mocking is used in unit tests to replace real dependencies (like external APIs, databases) with controlled, fake objects. This isolates the unit under test, makes tests faster, and allows testing of error conditions without external setup.

Testcontainers

What problem does Testcontainers solve for integration testing?

Answer

Testcontainers solves the problem of managing external dependencies for integration tests. It allows you to spin up real services (e.g., databases, message queues) in throwaway Docker containers, ensuring realistic, isolated, and consistent test environments.

pytest Advantages

List two key advantages of using pytest as your Python testing framework.

Answer

1. Simplicity: Easy to write tests with simple assert statements.
2. Powerful Fixtures: Reusable setup/teardown code.
3. Rich Assertion Rewriting: Provides detailed output on assertion failures.
4. Extensibility: Large plugin ecosystem.

unittest.mock.patch

How is unittest.mock.patch typically used in a test?

Answer

patch is used as a decorator or context manager to temporarily replace an object (e.g., a function, a class, a module attribute) with a mock object during the execution of a test. This allows you to control the behavior of dependencies.
