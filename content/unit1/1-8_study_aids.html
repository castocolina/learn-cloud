Flashcards: Testing Strategies

Question

Describe the concept of the Testing Pyramid.

Answer

The Testing Pyramid suggests having many fast, isolated unit tests at the base, fewer integration tests in the middle, and even fewer slow, comprehensive end-to-end tests at the top.

graph TD
    A["End-to-End Tests (Few, Slow)"]
    B["Integration Tests (More, Medium Speed)"]
    C["Unit Tests (Many, Fast)"]

    C --> B;
    B --> A;

Question

What is the primary goal of a unit test?

Answer

The primary goal of a unit test is to test individual functions or methods in isolation, providing fast and precise feedback on their correctness.

Question

When would you use mocking in your tests?

Answer

Mocking is used in unit tests to replace real objects (e.g., external APIs, databases) with controlled, fake objects, ensuring tests are fast, isolated, and predictable.

graph LR
    A["Component Under Test"] --> B{"Real Dependency?"};
    B -- Yes (Integration Test) --> C[Real Dependency];
    B -- No (Unit Test) --> D[Mock Object];
    D -- Simulates Behavior --> A;

Question

What problem do Testcontainers solve for integration testing?

Answer

Testcontainers provide lightweight, throwaway instances of real services (like databases or message queues) in Docker containers, enabling realistic, isolated, and consistent integration tests.

graph TD
    A[Test Code] --> B[Testcontainers Library];
    B --> C[Docker Daemon];
    C --> D["Ephemeral Docker Container (e.g., PostgreSQL)"];
    D -- "Provides Service" --> A;

Question

What is the main difference between unit and integration tests?

Answer

Unit tests focus on isolated components, while integration tests verify the interactions between multiple components or services.
