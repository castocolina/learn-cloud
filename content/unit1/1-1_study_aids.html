Flashcards: Development Environment & Tooling

Question

What is the primary purpose of pyenv in a Python development environment?

Answer

pyenv is used to manage multiple versions of Python on the same machine. It allows developers to easily switch between different Python versions on a per-project or global basis, ensuring that applications run against their intended interpreter version.

Question

What are two key advantages of using Poetry over a traditional requirements.txt and venv setup?

Answer

1. Deterministic Builds: Poetry uses a poetry.lock file to ensure everyone on a team (and the CI/CD pipeline) uses the exact same dependency versions.
2. Unified Tooling: It combines dependency management, virtual environment creation, and project packaging into a single tool, managed by the pyproject.toml file.

Question

What is the primary configuration file used by Poetry for project management?

Answer

The pyproject.toml file is the primary configuration file used by Poetry. It defines project metadata, dependencies, and other tool-specific configurations.

Question

Which IDE is recommended for Python cloud-native development in this course, and why?

Answer

Visual Studio Code (VS Code) is recommended due to its lightweight nature, powerful extensions, and seamless integration with Docker and cloud platforms, making it ideal for cloud-native development.

Question

Why is Docker considered a core part of the development loop in cloud-native development, even before deployment?

Answer

Docker ensures environment consistency and isolation. It guarantees that your application runs in the exact same environment on your local machine as it will in production, preventing "it works on my machine" issues and simplifying development.
