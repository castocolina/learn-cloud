Question 1

What is the primary purpose of pyenv in a Python development environment?

Answer

pyenv is used to manage multiple versions of Python on the same machine. It allows developers to easily switch between different Python versions on a per-project or global basis, ensuring that applications run against their intended interpreter version.

Question 2

What are two key advantages of using Poetry over a traditional requirements.txt and venv setup?

Answer

1. Deterministic Builds: Poetry uses a poetry.lock file to ensure everyone on a team uses the exact same dependency versions.

2. Unified Tooling: It combines dependency management, virtual environment creation, and project packaging into a single tool.

Question 3

What is the primary configuration file used by Poetry for project management?

Answer

The pyproject.toml file is the primary configuration file used by Poetry. It defines project metadata, dependencies, development dependencies, and other tool-specific configurations in a declarative format.

Question 4

What command creates a new Poetry project with the proper directory structure?

Answer

poetry new my-project-name creates a new project directory with the standard Python package structure, including a pyproject.toml file and basic project scaffolding.

Question 5

Why is a reproducible development environment crucial for cloud-native applications?

Answer

A reproducible environment eliminates "works on my machine" problems by ensuring consistency between development, testing, and production environments. This is critical for cloud-native apps that may run across different platforms and be deployed by CI/CD pipelines.
