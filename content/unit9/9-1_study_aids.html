Study Aids: Python E-Commerce Microservices Project

Key Takeaways
  • Microservices Architecture: Build loosely coupled services that communicate via REST APIs and message queues for better scalability and maintainability.
  • FastAPI Framework: Use FastAPI for high-performance REST API development with automatic OpenAPI documentation and async support.
  • Event-Driven Communication: Use message queues (RabbitMQ/Redis) to decouple services and enable asynchronous processing.
  • Infrastructure as Code: Use Terraform to provision cloud resources and Kubernetes to orchestrate containerized services.
  • Security Integration: Integrate HashiCorp Vault for secrets management and implement proper authentication/authorization patterns.

Question

What are the three core services in the Python E-commerce microservices project?

Answer

Product Service (REST API for managing products), Order Service (REST API for creating orders and publishing events), and Payment Worker (background service consuming order events and processing payments).

Question

Why is a message queue essential in this microservices architecture?

Answer

It enables asynchronous communication between services, decouples the Order Service from the Payment Worker, provides reliability through message persistence, and allows for better scalability and fault tolerance.

Question

What key technologies are integrated in this capstone project?

Answer

Python/FastAPI (services), PostgreSQL (data persistence), RabbitMQ/Redis (messaging), Kubernetes (orchestration), Terraform (IaC), Vault (secrets), GitHub Actions (CI/CD).

Question

How should database credentials be handled securely in this architecture?

Answer

Use HashiCorp Vault with the Vault Agent Injector pattern in Kubernetes. Services authenticate via their service account, retrieve database credentials dynamically at runtime, and never hardcode secrets in code or config files.

Question

What CI/CD pipeline features should be implemented for this project?

Answer

Automated testing (Pytest), code linting (Ruff), Docker image building and pushing to registry, SAST/SCA scanning (Bandit, pip-audit), and automated deployment to Kubernetes on main branch merges.
