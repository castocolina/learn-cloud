Study Aids: Core Backend Concepts

Key Takeaways
  • Go's Approach to OOP: Go achieves object-oriented principles through composition rather than inheritance, using a combination of structs (for state), methods (for behavior), and interfaces (for polymorphism).
  • SQL Database Interaction: Go provides two main approaches for relational databases: direct access with the standard database/sql package for maximum control, and higher-level abstraction with ORMs like GORM for faster development.
  • Security is Paramount: When using database/sql, it is critical to use parameterized queries (e.g., using $1 for PostgreSQL) to prevent SQL injection vulnerabilities.
  • NoSQL Integration: Go has robust, high-quality drivers for popular NoSQL databases, making it easy to integrate services like Redis for caching (using go-redis) or MongoDB for document storage (using the official mongo-go-driver).

Question

How does Go handle encapsulation and visibility (public/private) without access modifier keywords?

Answer

Go uses capitalization. A name (for a struct, field, function, etc.) is exported (public) if it begins with a capital letter. If it begins with a lowercase letter, it is unexported (private) and only visible within its own package.

Question

What is the primary purpose of the database/sql package in Go's standard library?

Answer

It provides a generic, standard interface for interacting with SQL or SQL-like databases. It is used in conjunction with a specific database driver (e.g., for PostgreSQL or MySQL) to execute queries, manage connections, and handle results.

Question

What is the main advantage of using an ORM like GORM compared to direct `database/sql` access?

Answer

The main advantage is increased developer productivity. GORM abstracts away raw SQL, allowing developers to perform database CRUD (Create, Read, Update, Delete) operations using Go structs and methods, which requires less boilerplate code.

Question

What is the most critical security practice when executing SQL queries with user-provided input in Go?

Answer

Using parameterized queries (e.g., `db.QueryRow("SELECT ... WHERE id = $1", userID)`). This prevents SQL injection attacks by ensuring that user input is treated as data, not as executable SQL code.

Question

Describe the common caching pattern implemented with Redis.

Answer

When requesting data, the application first checks if the data exists in the Redis cache. If it's a **cache hit**, the data is returned immediately. If it's a **cache miss**, the application queries the primary database, stores the result in Redis with an expiration time, and then returns the data.
