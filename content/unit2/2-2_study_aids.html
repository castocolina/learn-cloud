Study Aids: Overview & Foundational Concepts

Key Takeaways
  • Statically Typed & Compiled: Go's static type system catches errors at compile time, and it compiles to a single machine-code binary, leading to high performance and simple deployments.
  • Explicit Error Handling: Go favors explicit error handling using the `if err != nil` pattern, returning errors as values rather than using an exception-based system like Python's `try...except`.
  • Composition with Structs and Interfaces: Go uses `structs` to aggregate data (similar to Python's classes) and favors composition over inheritance. Its powerful implicit interfaces enable polymorphism and decoupled code.
  • Pointers for Efficiency: Go provides direct access to pointers, which is crucial for performance and allowing functions to modify data in place, a concept that is more abstracted in Python.

Question

How does Go's error handling philosophy differ from Python's?

Answer

Go uses an explicit, value-based error handling model. Functions return an `error` as their last return value, and the caller is expected to check `if err != nil`. This contrasts with Python, which uses an exception-based model with `try...except` blocks to handle errors.

Question

What is an implicit interface in Go?

Answer

A type implements an interface in Go simply by possessing all the methods defined by that interface. There is no `implements` keyword. This allows for decoupling, as functions can operate on interfaces without knowing the concrete type.

Question

What is a `struct` in Go, and what is its closest Python equivalent?

Answer

A `struct` is a composite type that groups together zero or more named fields of data. Its closest equivalent in Python is a `class`, particularly a `dataclass`, used for aggregating data attributes.

Question

What is the difference between a slice and an array in Go?

Answer

An **array** has a fixed size that is part of its type definition. A **slice**, on the other hand, is a more flexible, dynamically-sized view into the elements of an array. Slices are far more common in idiomatic Go code.

Question

Why are pointers used in Go function parameters?

Answer

Passing a pointer (`*MyType`) allows a function to modify the original value that was passed in. It is also more efficient for large structs, as it avoids copying the entire data structure, passing only a memory address instead.
