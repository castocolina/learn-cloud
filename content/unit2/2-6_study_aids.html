Study Aids: Building a RESTful API

Key Takeaways
  • Standard Library is Production-Ready: Go's built-in net/http package is powerful and robust enough to build production-grade REST APIs without any third-party dependencies.
  • Frameworks Accelerate Development: High-performance frameworks like Gin reduce boilerplate code for common tasks like routing, data binding, and middleware, which speeds up development for more complex applications.
  • JSON is a Core Task: APIs commonly communicate via JSON. This is handled by the standard encoding/json package for manual decoding/encoding or simplified by data binding features in frameworks.
  • It's a Trade-off: The choice between the standard library and a framework depends on the project's needs. The standard library offers simplicity and control, while frameworks offer a rich feature set and faster development speed.

Question

What is the role of http.ListenAndServe in Go's net/http package?

Answer

It starts an HTTP server that listens on a specified network address (e.g., ":8080") and uses a handler to process all incoming requests. It's a blocking call that runs until the server is shut down or an error occurs.

Question

What are two main advantages of using a framework like Gin over the standard net/http package?

Answer

1. Advanced Routing: Simplifies handling of path parameters (e.g., /users/:id) and grouping routes.
2. Middleware: Provides an easy way to add cross-cutting logic like logging, authentication, or error handling to requests.

Question

How do you read a JSON request body into a Go struct using the standard net/http package?

Answer

By using json.NewDecoder(r.Body).Decode(&myStruct), where r is the *http.Request and &myStruct is a pointer to the target struct instance you want to populate.

Question

What is the purpose of c.JSON(http.StatusOK, data) in the Gin framework?

Answer

It serializes the data (e.g., a struct or map) into a JSON string, sets the Content-Type header to application/json, and writes the response with the specified HTTP status code (200 OK in this case).

Question

When might you choose to use the standard net/http package instead of a framework like Gin?

Answer

For simple services where you want maximum control and minimal external dependencies, or for learning the fundamentals of how HTTP servers work in Go. It avoids adding a third-party dependency if the required features are basic.
