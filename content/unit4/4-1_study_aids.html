Study Aids: HashiCorp Consul

Key Takeaways
  • Consul is a Service Networking Tool: It solves three major challenges in microservices: service discovery, distributed configuration (KV store), and secure service-to-service communication (service mesh).
  • Service Discovery is Core: Consul's primary function is to allow services to find each other dynamically through DNS or an HTTP API. This eliminates the need for hardcoded IP addresses in a dynamic environment.
  • Dynamic Configuration with the KV Store: The distributed Key-Value (KV) store is ideal for managing dynamic configuration, such as feature flags or maintenance modes, across all services without requiring a redeploy.
  • Security with Consul Connect: Consul Connect, the built-in service mesh, automatically secures all inter-service traffic using mutual TLS (mTLS), providing both encryption and strong identity verification without requiring application code changes.

Question

What are the three main features of HashiCorp Consul?

Answer

1. Service Discovery: Helping services find each other.
2. Distributed Key-Value (KV) Store: For dynamic configuration.
3. Service Mesh (Consul Connect): For securing communication.

Question

In a Consul cluster, what is the difference between a "server" agent and a "client" agent?

Answer

Server agents (usually 3 or 5) form a consensus quorum to store and replicate the cluster's state. Client agents are lightweight, run on every application node, and act as a local proxy for service registration and health checking.

Question

How does Consul's service discovery work?

Answer

Services register themselves with a local Consul client. Other services can then query Consul (via its DNS or HTTP API) to get the current IP address and port of healthy instances of the services they need to connect to.

Question

What is Consul Connect?

Answer

It is Consul's built-in service mesh feature. It secures service-to-service communication by automatically providing and managing mutual TLS (mTLS) encryption and identity verification for all traffic, typically using sidecar proxies.

Question

What is a common use case for Consul's distributed Key-Value (KV) store?

Answer

Managing dynamic configuration, such as enabling or disabling feature flags across an entire fleet of services without requiring a redeployment. Applications can watch a key for changes and update their behavior in real-time.
